# Lab 13: General Network Troubleshooting

**Module:** M15 | **Course:** IT128 | **Date:** December 1, 2025

## Tools Used
- Windows CLI (ipconfig, nslookup, ping)
- pfSense
- NDG NetLab (WinOS + pfSense VM)

## What I Did
Performed systematic network troubleshooting using command line tools
and pfSense. Verified network configuration with ipconfig /all,
tested DNS resolution with nslookup, confirmed IPv4 and IPv6
connectivity with ping, and used pfSense as the reference point for
validating correct network settings including gateway, DNS, and
addressing.

## Key Concepts
- ipconfig /all — full adapter configuration verification
- nslookup — DNS resolution testing
- ping — IPv4 and IPv6 connectivity testing
- pfSense as gateway and DNS provider
- Systematic troubleshooting methodology
- IPv4 and IPv6 dual-stack verification

## Why This Matters
Network troubleshooting is the most practical skill in IT. Every
help desk call, NOC alert, and security incident starts with
verifying basic connectivity. This lab demonstrates a systematic
approach — verify config, test DNS, test connectivity — which is
the real-world methodology used by network administrators, SOC
analysts, and security engineers every day.
