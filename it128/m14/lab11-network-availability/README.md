**Module:** M14 | **Course:** IT128 | **Date:** November 30, 2025

## Tools Used
- pfSense
- NDG NetLab (WinOS + pfSense VM)

## What I Did
Accessed the pfSense web GUI and configured three network availability
services — NTP for time synchronization, SNMP for network monitoring,
and Syslog for centralized log collection. Verified each service was
configured correctly and functioning through the pfSense dashboard.

## Key Concepts
- NTP — Network Time Protocol for clock synchronization
- SNMP — Simple Network Management Protocol for device monitoring
- Syslog — centralized log collection and management
- pfSense web GUI administration
- Network availability and monitoring best practices

## Why This Matters
NTP, SNMP, and Syslog are the three pillars of network monitoring and
availability. Without accurate time synchronization, logs cannot be
correlated. Without SNMP, devices cannot be monitored centrally.
Without Syslog, events are lost. These services are required in every
enterprise network and are tested directly on the Network+ exam.
