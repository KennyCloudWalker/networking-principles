# Lab 06: Networking Commands and Tools

**Module:** M06 | **Course:** IT128 | **Date:** October 3, 2025

## Tools Used
- Windows Command Line
- ping, traceroute, ipconfig, arp
- NDG NetLab

## What I Did
Used command-line networking tools to test and troubleshoot network
connectivity. Verified adapter settings with ipconfig, tested
connectivity with ping, traced network paths with traceroute, and
inspected ARP tables to validate network operations and identify
potential issues.

## Key Concepts
- ipconfig — viewing and verifying adapter configuration
- ping — testing host reachability and latency
- traceroute/tracert — tracing network path hop by hop
- ARP table inspection — mapping IP to MAC addresses
- Practical network troubleshooting methodology

## Why This Matters
These command-line tools are the first line of defense in any network
troubleshooting scenario. Every network administrator, help desk
technician, and SOC analyst uses these daily. Mastery of these tools
is tested directly on the Network+ exam and expected in every
networking and IT support role.
